Underlying Causes & System Failures


The Therac-25’s failures stemmed primarily from software race conditions in its control program. AECL engineers had reused code from earlier machines (Therac-6 and Therac-20) that relied heavily on hardware safety interlocks. The Therac-25, however, removed many of these physical safeguards, assuming the software alone could prevent unsafe operation.



The software’s multitasking system allowed simultaneous updates from the operator console and treatment control unit. If an operator made rapid corrections — such as editing parameters mid-setup — the program could skip key safety checks. This resulted in the beam being activated at full power with the mechanical target arm in the wrong position, exposing patients to catastrophic electron doses instead of therapeutic X-rays.



Compounding the problem, the system provided misleading feedback: the screen displayed “Malfunction 54” or “No dose delivered,” implying harmless errors. There were no hardware dose monitors, no redundant interlocks, and no comprehensive testing of fault conditions. Each overdose occurred under slightly different operator conditions, obscuring the common root cause until forensic code reviews were performed in 1987.
