What Happened


On April 7, 2014, a serious security vulnerability in the OpenSSL cryptographic library was disclosed. The vulnerability now known as Heartbleed was a bug in the implementation of the TLS (Transport Layer Security) Heartbeat extension in versions of OpenSSL, the most common component used to encrypt online communications. Because OpenSSL was embedded in web servers, email systems, VPNs, and countless networked devices, the bug had silently exposed a large portion of the internet to potential data leaks for more than two years before being discovered.



It allowed anyone on the internet to craft a heartbeat request and get up to 64 KB of memory from a vulnerable server or client with no authentication. The memory disclosed via the leak could contain extremely sensitive information, such as private encryption keys, login credentials, session cookies, or anything else that happened to be in memory. With the attack leaving no trace in the logs, there is no reliable way to tell whether a given system had already been compromised, which caused uncertainty and urgency throughout the industry.

Major companies, hosting providers, and software vendors, in a race against time, have been patching affected systems, revoking and reissuing TLS certificates, and forcing password resets on millions of users since the bug became public. Emergency reviews of security have also been initiated by governments, banks, and operators of critical infrastructure. The incident showed how a single flaw in a widely relied upon open-source library could compromise global internet security and brought concerns about the lack of funding and oversight for critical security software used at massive scale.
