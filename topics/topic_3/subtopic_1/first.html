What Happened


On January 25, 2023, Microsoft experienced a large-scale service disruption that affected Microsoft 365 products such as Outlook, Teams, SharePoint, and OneDrive, as well as parts of the Azure cloud network. The issue began early in the morning (around 07:05 UTC), when users worldwide started reporting connection problems, login failures, and difficulties sending messages or accessing files. Microsoft quickly acknowledged the outage via its official service health dashboard and Twitter account, confirming that the problem was related to its internal networking infrastructure rather than an external cyberattack or system overload.

The disruption was traced back to a network configuration change made within Microsoft’s core wide-area network (WAN). A routine update intended to optimize network performance inadvertently caused a routing issue, leading to widespread packet loss and interruptions in communication between Microsoft’s data centers. Essentially, the network could no longer properly direct traffic between major regions, which cascaded into downtime across several dependent services. These types of incidents are especially impactful for Microsoft because many of its services share a common backbone and authentication system.



The outage lasted for roughly 90 minutes at its peak, though some users experienced lingering effects for several hours afterward. Microsoft engineers rolled back the problematic network update and restarted affected systems to restore connectivity. Once the fix was applied, most services gradually returned to normal, and Microsoft issued a formal post-incident report explaining the cause and preventive measures being implemented. While short-lived, the outage highlighted how a single misconfigured network command can have a global impact on millions of users relying on cloud services for communication and productivity.
